# Collaboration Guidelines

## Communication
- Use clear and concise language
- Provide context in communications
- Be respectful and constructive
- Ask questions when unclear

## Code Reviews
- Review code thoroughly but efficiently
- Provide specific, actionable feedback
- Focus on code quality and best practices
- Acknowledge good work and improvements

## Git Workflow
- Write descriptive commit messages
- Keep commits focused and atomic
- Use branches for features and fixes
- Rebase instead of merge when possible

## Documentation
- Document decisions and rationale
- Keep project documentation current
- Share knowledge and best practices
- Create guides for complex processes

## Meetings
- Prepare agenda in advance
- Stay focused on agenda items
- Follow up on action items
- Keep meetings efficient and productive
